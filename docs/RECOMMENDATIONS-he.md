## 1. שיפורים פנים-אפליקציה (ניווט · ייעול · "נוח בבוקר")
- ✅ **בוצע (·42)** **תצוגת בוקר / "היום"** — מסך ראשון (🌅) שמרכז: דורש-טיפול (אישורים + מלאי אדום), המשימות שלי, סטטוס הקמה. תלוי-תפקיד, צד-לקוח בלבד. *(הודעות שלא-נקראו כבר מטופלות בפופ-אפ הקיים בכניסה.)*
- ✅ **בוצע (·42)** **זכירת הדף האחרון** — נפתח איפה שעזבת באותו יום; ביום חדש נוחת על "היום".
- 🟢 **חיפוש-על (Command-K)** — כבר יש חיפוש; להרחיב אותו לקפיצה מהירה לכל מודול/קיבוץ/משימה ממקלדת אחת.
- 🟢 **קיצורי מקלדת** לניווט בין המודולים (1–9) ולפעולות נפוצות — ייעול לשולחן העבודה.
- 🟡 **ניווט בתוך הדפים** — "תפריט עוגן" צף בדפים הארוכים (קיבוצים/מלאי) לקפיצה בין סקציות; כותרות סקציה דביקות (sticky).
- 🟡 **טעינה מהירה** — skeleton/placeholder בזמן שליפת הנתונים (כבר חלקית קיים), ומטמון חכם יותר.

## 2. פיצרים נוספים
- 🟡 **עמוד פיתוח — שלב 2 (עריכה):** עדיפויות + סימון ספרינט הבא, דרך labels ב-GitHub (בלי Projects board). דורש טוקן עם הרשאת כתיבה.
- 🟡 **CHANGELOG/גרסאות EMS ביומן** — כפי שביקשת: ימי העלאת-גרסה של הפיתוח כאירועים ביומן, עם פירוט מה היה. מקור: ה-CHANGELOG שמתניה מתחיל לנהל (קובץ/GitHub Releases). *מחכה למקור.*
- 🟡 **ווידג'ט "מוכנות להעלאה לאוויר"** — מפרויקט הוולידציה (ראה סינרגיה).
- 🔵 **לוחות BI** — מאזן אנרגטי/פיננסי, בריאות מונים, מגמות צריכה (מפרויקט הגמר).
- 🟢 **הודעות + יומן משרד** — כבר בנויים/בדרך (טבלת messages, יומן office).

## 3. שירותי צד חשובים
- 🟡 **לוח שנה משרדי** — דרך Domain-Wide Delegation (מחכה לאדמין). זה פותח הוספת אירועים מתוך האפליקציה.
- 🟢 **GitHub proxy** — קיים; להשלים (תיקון GH_REPO/טוקן).
- 🔵 **אינטגרציית כלי הוולידציה** — להזרים את דוחות הוולידציה ל-Supabase ולהציג בתוך הדשבורד (ראה סינרגיה).
- 🔵 **Webhooks במקום polling** — GitHub/EMS דוחפים עדכון מיידי במקום רענון בכל טעינה.
- 🔵 **Push notifications** — התראות לטלפון (מלאי אדום, הודעה חדשה, קיבוץ דחוף).

## 4. נגישות / ייעול (לא לעיוורים — יעילות)
- 🟢 ניווט מקלדת מלא + פוקוס ברור.
- 🟢 "פעולות מהירות" (quick actions) בכל מסך — הפעולה הנפוצה במרחק קליק.
- 🟢 חיפוש-כל-מקום (סעיף 1) — להגיע לכל דבר בלי לדפדף.
- 🟡 הקטנת מספר המסכים/הקליקים למשימות יומיות (תיעוד ביקור, עדכון סטטוס).

## 5. תמיכה במובייל
- ✅ **כבר קיים** **bottom-nav** קבוע (אגודל מגיע לכל המודולים) ב-≤768px — לא נדרשת בנייה.
- 🟢 **יעדי-מגע גדולים** + טפסים נוחים לאצבע.
- 🟡 **Offline-first** — שהמערכת תיפתח ותעבוד גם בשטח בלי קליטה (ה-SW כבר מטמן; להרחיב לכתיבה-תורנית).
- 🟡 שיפור ה-PWA: אייקון/spinner/מסך-פתיחה (חלק קיים).

## 6. סינרגיה בין הפרויקטים שלך
**פרויקט הוולידציה (`New client in ems validation`)** — כלי Node שמשווה נתוני Legacy↔EMS ומפיק דוחות חריגות (8 בדיקות: מונים, תעריפים, צריכה, סולארי, חשבונות...).
- 🟡 **ווידג'ט מוכנות:** להזרים את דוחות הריצה ל-Supabase (`validation_runs` לפי קיבוץ) → אריח בדשבורד עם ✅/⚠️/❌ לכל קיבוץ ולכל בדיקה. רואים מי מוכן ל-go-live ומי דורש תיקון.
- 🟡 **Drill-down + תיקון בקליק:** לחיצה על חריגה → פאנל עם הדוח + הצעת תיקון (תבנית אקסל למונה, שיוך קבוצת תעריף).
- 🔵 **Trigger אוטומטי:** סימון "מוכן ל-go-live" בדשבורד → מריץ ולידציה → שומר תוצאה → מעדכן סטטוס. שער איכות לפני העלאה.

**פרויקט הגמר (BI Power-BI למנהל המחלק)** — מחסן נתונים ממדי (snowflake) מעל ה-EMS: מאזן אנרגטי/פיננסי, ToU, סולארי, בריאות מונים.
- 🔵 **לוחות BI בדשבורד:** מאזן פיננסי (₪), אובדן רשת (%+₪), יעילות סולארית, מונים שלא משדרים — בדיוק ה-KPIs שמנהל מחלק רוצה.
- 🟡 **תוויות עדכניות (freshness):** "נתונים מעודכנים עד [תאריך]" + סימון "היום (חלקי)" — אמון בנתונים.
- 🟡 **טבלת יעדי-KPI** נפרדת (סף ירוק/צהוב/אדום ללא קוד קשיח) — ל"ביקורים לרבעון", "הזמנות תוך 5 ימים" וכו'.
- 🔵 **מודל ממדי ל-Supabase** (אם נגדל): קיבוץ כ-hub, ביקורים/נוכחות/הזמנות כ-facts, SCD-2 לסטטוסים — מחזק שלמות נתונים בלי לסבך.
